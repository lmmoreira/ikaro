Protótipo — GUEST: Agendar Serviço

Fluxo de 4 passos. Todos os componentes já existem em apps/web/components/booking/ (M12-S07). Este protótipo é para revisão de UX — nenhum código novo é necessário para o caminho do guest.

  • Tela 0 — Hotsite (ponto de entrada) EXISTS: HeroModule + ServiceListModule + BookingCtaModule
  • Passo 1 — Escolha os serviços EXISTS: ServiceSelectionStep
  • Passo 2 — Calendário e horário EXISTS: AvailabilityCarousel + SlotPicker
  • Passo 3 — Seus dados EXISTS: PersonalInfoStep
  • Passo 4 — Confirmar agendamento EXISTS: ConfirmationStep

Estados alternativos (unhappy paths)

Cada tela abaixo é uma variante de um dos 4 passos acima, mostrando um estado de erro, vazio, carregamento ou sucesso. Nenhuma é uma rota nova — todas usam os mesmos componentes (já existentes), apenas em outro estado.

  • Passo 1b — Erro: endereço de coleta obrigatório EXISTS: ServiceSelectionStep + AddressFields
  • Passo 2b — Carregando disponibilidade EXISTS: AvailabilityCarousel (loading)
  • Passo 2c — Erro ao carregar disponibilidade ⚠ sem botão de retry
  • Passo 2d — Sem horários disponíveis (totalmente lotado) ⚠ sem mensagem explicativa
  • Passo 2e — Horário indisponível (409 ao confirmar) EXISTS: SlotPicker (step2-error)
  • Passo 3b — Erro de validação (e-mail inválido) EXISTS: PersonalInfoStep (personal-info-error)
  • Passo 3c — Estados do upload de fotos (enviando / enviada / erro) ⚠ item com erro não tem "Remover"
  • Passo 4b — Enviando agendamento EXISTS: ConfirmationStep (submitting)
  • Passo 4c — Erro ao enviar agendamento EXISTS: ConfirmationStep (confirmation-error)
  • Passo 4d — Solicitação enviada (sucesso) EXISTS: ConfirmationStep (confirmation-success)
O que este protótipo valida:
✅ Sequência de 4 passos faz sentido para o guest?
✅ Pickup address no passo 1 (antes do calendário) é o lugar certo?
✅ Formulário de dados pessoais no passo 3 está claro e completo?
✅ Resumo do passo 4 contém todas as informações necessárias para confirmar?
✅ Tokens de branding (--ba-*) são os mesmos que o hotsite usa?
✅ Passo 2d (sem horários) — aceitável sem mensagem, ou precisa de copy explicativo?
✅ Passo 2c (erro ao carregar) — precisa de um botão de retry?
✅ Passo 3c (erro no upload) — precisa de "Remover"/tentar novamente no item com erro?
