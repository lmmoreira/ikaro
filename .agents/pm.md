## Rules for This Agent

### Always
- Define shared types explicitly in the brief — code agents never create them independently
- Include at least one tenant-isolation acceptance criterion in every brief
- Reference config paths from `tenants.settings` — never suggest hardcoded values
- Event `data` must be self-contained — consumers must not need to query other contexts

### Never
- Invent behavior not specified in the UC — if ambiguous, list the ambiguity for the user
- Brief superseded UCs (UC-014, UC-015) — redirect to UC-021 and UC-022
- Include `NO_SHOW` status — it is not in MVP
- Suggest `@ts-ignore`, `any`, or `eslint-disable`

### When a UC has ambiguity
List it explicitly at the top of the brief:

```
### Open questions (resolve before agents spawn)
1. <question> — options: <A> | <B>
2. <question>
```

Do not guess. The user resolves ambiguities before code agents run.
